# 项目进度审计技能（Project Progress Auditor）

## 适用场景
- 用户要求“检查项目进度”“分析完成情况”“继续完善项目并按功能测试推进”。
- 需要把“功能实现”和“测试门禁”绑定成固定流程。

## 输入
- 仓库根路径（默认当前目录）。
- 可选：当前迭代目标、优先级、风险约束。

## 标准流程
1. 基线扫描（排除构建产物）
   - `git status --short`
   - `rg --files -g '!target/**'`
   - `rg -n --glob '!target/**' "TODO|FIXME|SKILL\\.md|AGENTS\\.md|skill_router|planner|lifecycle|executor|registry"`
2. 建立完成度矩阵
   - 已完成：有实现+有测试或可执行证据。
   - 进行中：有实现但缺测试/缺集成。
   - 未开始：仅有计划或需求。
3. 选择“一个最小可交付功能”进行实现。
4. 功能完成后立即测试（门禁）
   - 先跑该功能最相关测试。
   - 再跑最小回归（例如 `cargo test`）。
   - 测试失败则修复，禁止进入下一功能。
5. 输出阶段结论
   - 本功能改动点
   - 测试命令与结果
   - 下一功能候选（不自动开始）

## Rust 项目默认测试门禁
- 单测/集成：`cargo test`
- 指定集成测试：`cargo test --test skill_router_pipeline`

## 输出模板
- 进度快照：已完成/进行中/未开始
- 本次功能：目标、改动、风险
- 测试记录：命令、通过/失败、失败摘要
- 下一步：仅在当前测试通过后执行

## 约束
- 不在未通过测试时推进下一功能。
- 优先小步迭代，每步可验证、可回滚。
- 扫描时默认排除 `target/`，避免噪音。
