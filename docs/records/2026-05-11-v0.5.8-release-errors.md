# v0.5.8 发布收尾 — 错误记录 / Release Closeout Errors

> 撰写日期：2026-05-11
> 对应版本：v0.5.8

---

## 错误 1：PR 冲突导致发布未完成

- **现象**：之前的 session 推送了 tag 就宣布发布成功，但 PR #4 仍有冲突未合并到 main
- **根因**：`claude/lucid-engelbart-8f807d` 分支与 `main` 分支各自有修改（main 有 SerpAPI 多 Key + DeepSeek 超时修复），产生 6 个冲突文件
- **修复**：逐一解决冲突后合并到 main，重新指向 tag
- **教训**：发布前必须检查 PR mergeable 状态，tag 必须指向 main 的合并提交

## 错误 2：`cargo fmt` 合并后格式失败

- **现象**：合并冲突解决后运行 `cargo fmt --check` 发现 mod.rs、search.rs、settings.rs 有多余空行和缩进问题
- **根因**：Python 脚本自动解决冲突时未保留标准 Rust 格式（`}` 后多一个空行）
- **修复**：运行 `cargo fmt` 自动修正
- **教训**：冲突解决后务必运行 `cargo fmt --check`

## 错误 3：GitHub TLS 连接失败

- **现象**：推送 docs commit 到 GitHub 时反复出现 `SSL routines::unexpected eof while reading`
- **根因**：GitHub 网络不稳定，偶发 TLS 断开
- **修复**：先推送到 Gitee 成功，GitHub 待网络恢复后重试
- **教训**：使用 `origin` 多远程推送时，任一失败会导致整体失败，可单独推送到各远程

## 错误 4：文档合集未拆分（已修复）

- **现象**：将修改记录、未完成、复盘合并为一份 closeout 文档
- **根因**：忽略了之前已经明确要求的"三份文档分开"
- **修复**：删除合集文档，拆分为三个独立文件（changelog / retrospective / errors）
- **教训**：按用户已建立的规范执行，不要自创格式
