### 可改进的

1. **发布流程应先确认** — 之前的 session 以为推送了 tag 就是发布成功，实际 PR 有冲突未合并到 main。下次应先检查 PR 状态再做发布声明

### 关键教训

- **Tag 在分支上 ≠ 发布完成**：标签必须指向 `main` 分支的合并提交才是正式发布
- **PR 合并前必须检查冲突**：`gh pr view --json mergeable` 可提前发现冲突
- **Release 资产会过期**：如果发布后发现需要补充变更，必须重新构建并上传安装包
