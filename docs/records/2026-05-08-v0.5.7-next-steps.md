## 用户需操作事项

1. **重新构建运行**：修改了代码，需要重新编译启动才能生效
   ```bash
   cargo build --release
   ```

2. **检查设置页**：启动后打开 `/settings` 页面：
   - 确认 SerpAPI Key 配置正确（已在日志中确认有效）
   - 确认 DeepSeek 配置已设为 Primary（日志中曾看到 `[failover] DeepSeek succeeded`）
   - 如果 DeepSeek 仍然在 fallback 位置，AI 自动提升逻辑会将其 Promotion
   - Big/Firecrawl 等失效 key 无需操作，已自动静默跳过
