# v0.5.8 多平台补齐发布 — 错误记录 / Backfill Errors

> 撰写日期：2026-05-11
> 目标版本：v0.5.8

---

## 错误 1：手动触发后上传资产失败（缺少 tag）

- **现象**：构建与打包均成功，但 `Upload release asset` 步骤失败。
- **报错**：`GitHub Releases requires a tag`
- **根因**：`workflow_dispatch` 触发时 `action-gh-release` 未自动获取目标 tag。
- **修复**：在发布步骤显式传入 `tag_name`（手动触发取输入 tag，标签触发取 `github.ref_name`）。

## 错误 2：重发时同名资产冲突

- **现象**：同一标签重复补发时，上传步骤失败。
- **根因**：旧资产同名，默认不覆盖。
- **修复**：增加 `overwrite_files: true`，允许覆盖同名文件。

## 错误 3：监控阶段网络波动

- **现象**：`gh run view/watch` 出现 TLS handshake timeout / EOF。
- **根因**：GitHub API/日志下载链路瞬时网络波动。
- **修复**：重试轮询与分步读取，最终获取完整状态和结果。

---

## 教训

1. 历史标签补发必须同时处理“tag 解析”和“同名覆盖”两个问题。
2. 线上发布监控需要具备重试策略，不能依赖单次 API 成功。
