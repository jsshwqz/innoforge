## 二、详细变更 / Detailed Changes

### A. 代码重构（消除重复 + unsafe 消除）

#### A1. Jaccard 相似度函数提取为共享工具 / Shared Jaccard util

**问题**：`jaccard` 相似度算法在 3 个文件中各有一份实现，逻辑重复。
- `src/pipeline/steps/similarity.rs` — `fn jaccard`
- `src/pipeline/steps/contradiction.rs` — `fn token_jaccard`
- `src/pipeline/steps/prior_art_cluster.rs` — `fn jaccard_similarity`

**修复**：
- 新增 `src/pipeline/steps/text_util.rs`，导出泛型 `pub fn jaccard<T, S>(a: &HashSet<T, S>, b: &HashSet<T, S>) -> f64`
- `similarity.rs` / `contradiction.rs` / `prior_art_cluster.rs` 移除本地实现，统一 `use crate::pipeline::steps::text_util::jaccard;`
- `mod.rs` 新增 `pub mod text_util;`
- 保留 `idea.rs` 中的 `jaccard_trigram`（三元组语义不同：空对空返回 1.0，而通用版返回 0.0）

#### A2. `static mut` → `Mutex`（消除 unsafe）/ Eliminate unsafe in FFI

**问题**：`src/lib.rs` 使用 `static mut SERVER_HANDLE` + `unsafe` 块管理移动端 FFI 单例服务器句柄，存在未定义行为风险。

**修复**：
```rust
// 之前（unsafe）
static mut SERVER_HANDLE: Option<(JoinHandle<()>, oneshot::Sender<()>)> = None;
// innoforge_start_server() 内: unsafe { SERVER_HANDLE = Some(...) }

// 之后（safe）
static SERVER_HANDLE: std::sync::Mutex<Option<(JoinHandle<()>, oneshot::Sender<()>)>>
    = std::sync::Mutex::new(None);
// innoforge_start_server() 内: SERVER_HANDLE.lock().unwrap().replace(...)
// innoforge_shutdown_server() 内: SERVER_HANDLE.lock().unwrap().take()
```
- 移除全部 `unsafe` 块和 `#[allow(static_mut_refs)]` 标注
- 选择 `std::sync::Mutex` 而非 `OnceLock`：FFI 需要 mutable access（start/shutdown）

### B. 路径可移植性 / Path Portability

**问题**：多处硬编码 `C:\Users\Administrator\...` 路径，导致项目无法在其他机器或非默认用户目录运行。

#### B1. `src/routes/mod.rs` — `find_gemini_cli()`

```diff
  let candidates = [
-     r"C:\Users\Administrator\AppData\Roaming\npm\gemini.cmd",  // ❌ 硬编码
      r"C:\Program Files\nodejs\gemini.cmd",
      r"C:\Program Files (x86)\nodejs\gemini.cmd",
      "gemini.cmd",
      "gemini",
  ];
  // ... 保留 USERPROFILE 环境变量自动检测 + PATH 查找
```

#### B2. `src/routes/upload.rs` — Python 解释器路径

4 处 `let python = r"C:\Users\Administrator\AppData\Local\Programs\Python\Python313\python.exe";` 替换为 `find_python()` 调用：

```rust
fn find_python() -> String {
    // 1. 优先 PYTHON 环境变量
    // 2. python3（Unix）
    // 3. python（Windows / 通用）
    // 4. 回退到硬编码路径（仅作为最后手段）
}
```

### C. `.env` 密钥脱敏 / Secret Redaction

**问题**：`.env` 包含真实 API 密钥（DeepSeek / Gemini / Zhipu / SerpAPI 等），存在泄露风险。

**修复**：
- 全部真实密钥替换为 `your-xxx-key-here` 占位符
- 保留非敏感配置（`AI_BASE_URL`、`AI_MODEL`、`GEMINI_CLI_ENABLED=false`）
- 重写为 UTF-8 编码（原为 UTF-16 LE BOM，导致 `cat`/`grep` 无法读取）
- 结构与 `.env.example` 对齐，分段标注

> **注意**：本次仅脱敏，未轮换密钥。建议后续在服务商控制台轮换被暴露的密钥。
