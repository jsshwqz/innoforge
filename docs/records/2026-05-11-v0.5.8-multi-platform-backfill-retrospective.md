# v0.5.8 多平台补齐发布 — 复盘 / Backfill Retrospective

> 撰写日期：2026-05-11
> 目标版本：v0.5.8

---

## 做得好的

1. **定位准确**：第一时间区分“构建成功”与“上传失败”，避免误判为编译问题。  
2. **修复最小化**：仅改 `release.yml`，不引入业务代码变更，发布风险低。  
3. **闭环执行**：修复后立即重跑并验证资产列表，直到全平台资产落地。  
4. **可复用性提升**：流程已支持历史 tag 补发，后续版本可直接套用。  

## 可改进的

1. **预防性校验不足**：首次增加 `workflow_dispatch` 时应同步补 `tag_name` 与覆盖策略，减少试错轮次。  
2. **发布可观测性**：可增加“发布前自检步骤”，提前打印 `tag_name` 与上传目标文件。  

## 固化动作

1. 保留当前 `workflow_dispatch + tag_name + overwrite_files` 作为标准发布模板。  
2. 后续凡是“历史版本补发”，统一走同一路径并保留 run 链接与资产清单。
