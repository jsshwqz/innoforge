# 2026-07-13 AI 提示词注入边界修复计划

## 背景

`/api/ai/chat` 接受客户端提供的历史消息；多模态路径将其 `role` 原样放入上游消息数组，普通路径也会原样传给多轮聊天。这允许恶意客户端伪造 `system` 消息。与此同时，OA 讨论将审查意见、既有分析、历史讨论和最新意见直接插入系统或用户提示词；专利记录与联网搜索材料也会作为系统上下文传递。外部材料中的伪指令可能与固定任务规则混淆。

## 范围

仅修改 `src/routes/ai.rs`（包含该文件内的单元测试）。不新增 crate、不修改公开 API、路由、数据库 schema 或前端模板。

## 实施步骤

1. 在 AI 路由模块中提供统一的外部材料边界工具：使用固定说明和 `<user_input>` 包装，并转义 `&`、`<`、`>`，使材料无法伪造闭合标签或系统段落。
2. 对聊天历史实施角色白名单：仅接受 `user` 和 `assistant`；拒绝其他角色，避免客户端提升为 `system`。对早期历史压缩结果以非系统消息保留，禁止摘要成为新的系统指令。
3. 将专利记录、联网搜索结果、OA 审查意见、既有分析、历史讨论和最新用户意见作为边界化参考材料插入，保留原始文本，不做截断。
4. 添加纯函数回归：伪造闭合标签会被转义；`system` 等非白名单历史角色被拒绝；合法历史保持原有顺序和内容。
