# 📁 Patent Hub 文档系统说明

**本文档解释文档系统的组织方式和使用方法**

---

## 🎯 文档系统结构

```
patent-hub-backup/
│
├── .kiro/
│   └── agent.md             ← AI 助手入口（首次接触必读）
│
├── DOCS_INDEX.md            ← 文档导航总入口（人类用户起点）
├── PROJECT_CONTEXT.md       ← 完整上下文（最重要！）
├── README.md                ← 项目介绍
│
├── docs/                    ← 用户文档目录
│   ├── QUICK_START.md
│   ├── INSTALL.md
│   ├── ARCHITECTURE.md
│   ├── API.md
│   └── ...
│
├── SECURITY_FIXES.md        ← 安全修复详情
├── VERIFICATION_REPORT.md   ← 验证测试报告
│
└── *.md                     ← 历史修复报告（按日期参考）
```

---

## 📖 阅读顺序规则

### 对于 AI 助手（Kiro/Claude 等）

**自动触发**: `.qwen/QWEN.md` 会在以下情况被读取：
1. 用户打开新的对话会话
2. 用户询问项目相关问题
3. 用户要求修改代码

**阅读顺序**:
```
1. .qwen/QWEN.md             (自动读取)
   ↓
2. PROJECT_CONTEXT.md        (按指引阅读)
   ↓
3. 其他相关文档              (按问题类型选择)
```

---

### 对于人类用户

**入口**: `DOCS_INDEX.md`

**阅读顺序**:
```
1. DOCS_INDEX.md             (文档导航)
   ↓
2. PROJECT_CONTEXT.md        (完整上下文)
   ↓
3. README.md                 (项目介绍)
   ↓
4. 其他文档                  (按需阅读)
```

## 🔑 关键文档说明

### 1. `.kiro/agent.md` - AI 助手入口
**位置**: `.kiro/agent.md`  
**作用**: 指导 AI 助手如何阅读文档  
**触发**: 自动（Kiro、Qwen Code 等工具会读取）

**内容**:
- 首次接触项目的阅读顺序
- 根据用户问题选择文档的指南
- 文档优先级说明

---

### 2. `DOCS_INDEX.md` - 文档导航
**位置**: `DOCS_INDEX.md`  
**作用**: 人类用户的文档起点  
**触发**: 用户手动打开

**内容**:
- 快速开始阅读顺序
- 文档分类索引
- 按场景阅读指南
- 关键文档说明

---

### 3. `PROJECT_CONTEXT.md` - 完整上下文
**位置**: `PROJECT_CONTEXT.md`  
**作用**: 项目技术决策和设计要求  
**触发**: 按指引阅读

**内容**:
- 功能清单和实现状态
- 关键技术决策（6 项）
- API 端点完整清单
- 已知问题和解决方案
- 测试验证指南

## 🔄 文档发现机制

### 自动发现（AI 助手）

1. **Kiro/Qwen Code** 会读取 `.kiro/agent.md`
2. 根据 `agent.md` 指引读取其他文档
3. 理解项目上下文后回答问题

### 手动发现（人类用户）

1. 打开 `DOCS_INDEX.md`
2. 按导航阅读相关文档
3. 查找所需信息

---

## 📝 文档维护指南

### 添加新文档时

1. **更新 `DOCS_INDEX.md`**
   - 在对应分类中添加链接
   - 更新"按场景阅读"指南（如需要）

2. **更新 `.kiro/agent.md`**（如需要）
   - 如果新文档很重要，添加阅读指引

3. **更新 `PROJECT_CONTEXT.md`**（如需要）
   - 如果涉及技术决策变更

---

### 删除旧文档时

1. **更新 `DOCS_INDEX.md`**
   - 从索引中移除链接

2. **检查引用**
   - 确保其他文档没有引用已删除文档

---

### 文档版本控制

所有文档应在底部包含：
```markdown
**最后更新**: YYYY-MM-DD  
**维护者**: Patent Hub Team  
**文档版本**: X.X
```

## 📞 常见问题

### Q: 为什么需要两个入口文档？

**A**: 
- `.kiro/agent.md` → AI 助手自动读取
- `DOCS_INDEX.md` → 人类用户手动打开

两者格式和内容侧重点不同。

### Q: 如何确保 AI 助手读取最新文档？

**A**: 
1. 在 `.qwen/QWEN.md` 中明确列出必读文档
2. 在 `PROJECT_CONTEXT.md` 中记录最后更新日期
3. AI 助手会优先读取最新文档

### Q: 文档太多找不到怎么办？

**A**: 
1. 先读 `DOCS_INDEX.md`
2. 按"快速开始"顺序阅读
3. 使用搜索功能查找关键词
