## 手动创建 Release 步骤

由于 GitHub Actions 自动构建遇到问题，建议手动创建 Release：

### 步骤 1: 访问 Release 页面
打开浏览器，访问:
```
https://github.com/jsshwqz/patent-hub/releases/new
```

### 步骤 2: 选择 Tag
- 在 "Choose a tag" 下拉框中选择 `v0.1.0`
- 或者输入 `v0.1.0` 并选择 "Create new tag: v0.1.0 on publish"

### 步骤 3: 填写 Release 信息
- **Release title**: `Patent Hub v0.1.0 - 专利检索分析系统`
- **Description**: 复制 `RELEASE_NOTES_v0.1.0.md` 文件的内容

### 步骤 4: 上传文件
点击 "Attach binaries by dropping them here or selecting them"，上传：
- `patent-hub-v0.1.0-windows-x86_64.zip` (已构建，位于项目根目录)

### 步骤 5: 发布
- 确保 "Set as the latest release" 已勾选
- 点击 "Publish release" 按钮

## 多平台构建 (可选)

如果需要 Linux 和 macOS 版本，可以：

### 方法 1: 本地构建 (需要对应系统)
```bash
# Linux
cargo build --release
cd release-package && tar czf ../patent-hub-v0.1.0-linux-x86_64.tar.gz *

# macOS
cargo build --release
cd release-package && tar czf ../patent-hub-v0.1.0-macos-x86_64.tar.gz *
```

### 方法 2: 使用 Docker (推荐)
```bash
# Linux 版本
docker run --rm -v $(pwd):/app -w /app rust:latest cargo build --release

# 然后打包
cd release-package && tar czf ../patent-hub-v0.1.0-linux-x86_64.tar.gz *
```

### 方法 3: 修复 GitHub Actions
编辑 `.github/workflows/release.yml`，确保：
1. 使用 `softprops/action-gh-release@v1`
2. 正确配置构建步骤
3. 测试工作流
